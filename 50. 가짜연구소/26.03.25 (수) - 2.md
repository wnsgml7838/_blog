## 2. 타겟 유저 & 문제 정의

### 2.1 Primary Persona: 독서 모임 운영자 (모임장)

> "독서 모임을 만들고 싶은데, 준비하는 데 드는 시간과 노력이 너무 커서 시작을 못 하거나 지속이 어려운 사람"

- 월 1회 이상 독서 모임을 운영하거나 운영하려는 사람
- 모임 세팅, 토론 준비, 기록 관리까지 혼자 도맡는 역할
- 카카오톡, 노션, 줌, 구글 캘린더 등 여러 툴을 파편적으로 사용 중

### 2.2 페인포인트

| #   | 페인포인트        | 상세                                                         | AI 해결 여부 |
| --- | ------------ | ---------------------------------------------------------- | -------- |
| ①   | **세팅 피로도**   | 노션 템플릿 찾기 → 커스텀 → 안 써본 멤버 온보딩… "모임 운영"이 아니라 "세팅"에 에너지를 다 씀 | UX로 해결   |
| ②   | **토론 준비 부담** | 모임장이 매번 발제문/토론 질문을 직접 만들어야 하며, 책 한 권당 1~2시간 소요. 모임장만 번아웃   | ✅ AI 필수  |
| ③   | **기록 휘발**    | 모임에서 좋은 이야기가 나와도 기록이 안 되거나 흩어져서 나중에 찾을 수 없음                | ✅ AI 필수  |

### 2.3 AI가 풀어야 하는 이유

- 페인포인트 ①은 올인원 UX로 해결 가능
- 페인포인트 ②③은 사람이 수동으로 하면 지속 불가능한 영역
- 토론 질문 자동 생성, 모임 대화 자동 정리는 AI 없이는 현실적으로 불가능
- 이 두 기능이 ReadSync가 "또 다른 노션 클론"이 아닌 이유

#### P0 (MVP 필수) — 토론 발제/질문 자동 생성

|항목| 내용                                                  |
|---|---|
|**해결하는 문제**| 모임장이 매번 토론 질문을 직접 만드는 번아웃                           |
|**동작 방식**| 사용자가 읽은 범위(챕터/페이지)를 입력 → 해당 구간에 대한 토론 질문 3~5개 자동 생성 |
|**질문 유형**| "내용 이해 확인", "개인 경험 연결", "비판적 사고" 등 카테고리별 분류 제공      |
|**필요 모델**| LLM — Qwen3.5-9B (Apache 2.0)                       |
|**레이턴시 목표**| ~5~10초 (Cold start 포함 시 +15초)                       |

#### P0 (MVP 필수) — 모임 회의록 자동 정리 (STT + 요약)

| 항목          | 내용                                                                            |
| ----------- | ----------------------------------------------------------------------------- |
| **해결하는 문제** | 모임 후 기록이 휘발되는 문제                                                              |
| **동작 방식**   | 회의 녹음 → STT 텍스트 변환 → 화자 분리 → LLM이 주제별 요약 + 핵심 발언 + 액션 아이템 정리                  |
| **처리 방식**   | 비동기 — 회의 종료 후 2~3분 내 알림으로 회의록 제공                                              |
| **필요 모델**   | STT: faster-whisper (large-v3) / 화자 분리: pyannote 또는 WhisperX / 요약: Qwen3.5-9B |

#### P1 (MVP 직후) — 개인 독서 노트 요약/인사이트 추출

|항목| 내용                                            |
|---|---|
|**해결하는 문제**| 쌓인 노트에서 패턴과 인사이트를 수동으로 찾기 어려움                 |
|**동작 방식**| 개인 노트 분석 → 가장 많이 언급한 키워드, 반복 관심 주제, 한 줄 요약 추출 |
|**필요 모델**| Qwen3.5-9B (동일 모델)                            |

#### P2 (확장) — 맞춤 도서 추천

| 항목          | 내용                               |
| ----------- | -------------------------------- |
| **해결하는 문제** | 다음 책 선정에 시간 소요                   |
| **동작 방식**   | LLM + 외부 도서 DB API 조합 (아래 상세 참고) |
| **필요 모델**   | Qwen3.5-9B+ 추후 임베딩 모델 (BGE-M3 등) |

**왜 LLM 단독 추천이 아닌가:**

8B 모델은 학습 데이터에 포함된 책만 알고, 한국어 도서 커버리지가 특히 낮음. 존재하지 않는 책을 생성하는 할루시네이션 위험이 높아 LLM 단독 추천은 신뢰도 확보가 어려움. Claude/GPT-4급에서도 도서 추천 할루시네이션이 발생하는 영역.

**해결 방식: LLM + 외부 도서 DB API 조합 아키텍처**

```
사용자 독서 노트 / 독서 이력
  → Qwen3.5-9B: 키워드·주제·취향 추출
  → 추출된 키워드로 외부 도서 DB API 검색 (실제 존재하는 책 후보 확보)
  → Qwen3.5-9B: 후보 중 사용자 취향에 맞는 책 선별 + 추천 이유 생성
  → 프론트에 추천 결과 + 구매 링크 제공
```

- LLM 역할: ① 키워드/취향 추출, ② 최종 선별 + 추천 이유 자연어 생성
- 외부 API 역할: 실제 존재하는 도서 데이터 제공 (할루시네이션 원천 차단)
- 부가 효과: 알라딘 링크 제공으로 구매 전환 가능

**활용 가능한 외부 도서 DB API:**

|API|특징|추천에 유용한 기능|제한|
|---|---|---|---|
|**알라딘 Open API** (추천)|검색, 상품 상세, 베스트셀러/신간/편집자 추천 리스트 제공. JSON 지원. 카테고리 매우 세분화|베스트셀러·신간·편집자 추천 리스트로 큐레이션 가능, 세분화된 카테고리로 장르별 필터링|1일 5,000회. TTB 키 발급 필요 (1~2일 소요)|
|**카카오 책 검색 API**|Daum 책 서비스 기반. JSON 응답, CORS 이슈 적음|제목/저자/ISBN 검색. 프론트엔드 연동 간편|큐레이션 리스트 미제공|
|**네이버 책 검색 API**|Client ID/Secret 기반. JSON 지원|제목/저자 검색|표지 이미지 크기 작음 (82x116)|

**Phase별 진화 경로:**

- Phase 2 초기: 알라딘 API 검색 + Qwen3 키워드 추출/추천 이유 생성
- Phase 3: 임베딩 모델(BGE-M3) 추가 → 사용자 노트 임베딩 기반 유사도 추천으로 고도화

### 4.3 AI 파이프라인 상세

#### 파이프라인 A: 토론 질문 생성

```
사용자 입력 (책 제목 + 읽은 범위 + 장르)
  → 백엔드 API
  → RunPod Serverless (Qwen3.5-9B)
  → 질문 3~5개 JSON 반환
  → 프론트에 렌더링
```

- 레이턴시 목표: ~5~10초 (Cold start 포함 시 +15초)

#### 파이프라인 B: 회의록 자동 정리

```
회의 종료 → 녹음 파일 업로드
  → RunPod Worker Step 1: faster-whisper (STT + 타임스탬프)
  → RunPod Worker Step 2: 화자 분리 (pyannote)
  → RunPod Worker Step 3: Qwen3.5-9B (주제별 요약)
  → 구조화된 회의록 JSON 반환
  → DB 저장 + 프론트 표시
```

- 비동기 처리: 회의 종료 후 2~3분 내 알림 제공

#### 파이프라인 C: 노트 인사이트 추출 (P1)

```
"인사이트 분석" 버튼 클릭
  → 해당 책의 개인 노트 전체 수집
  → RunPod Serverless (Qwen3.5-9B)
  → 키워드, 반복 주제, 한 줄 요약 반환
```

#### 파이프라인 D: 맞춤 도서 추천 (P2)

```
"다음 책 추천" 요청
  → 백엔드: 사용자 독서 이력 + 노트 데이터 수집
  → RunPod Serverless (Qwen3.5-9B): 키워드·주제·취향 추출
  → 백엔드: 추출된 키워드로 알라딘 Open API 검색 + 리스트 조회
  → RunPod Serverless (Qwen3.5-9B): 후보 중 최종 선별 + 추천 이유 생성
  → 추천 결과 + 알라딘 링크 반환
```

- LLM은 추천 "로직"이 아닌 키워드 추출 + 추천 이유 "설명" 역할
- 실제 도서 데이터는 알라딘 API가 제공 → 할루시네이션 원천 차단

## 5. MVP 범위 & 로드맵

### 5.1 MVP (Phase 1) — 핵심 루프 완성

> **목표:** 모임장이 ReadSync 하나로 모임을 준비하고, 진행하고, 기록할 수 있다

| 기능              | 상세                            | AI  |
| --------------- | ----------------------------- | --- |
| 모임 생성 & 책 등록    | 책 제목/저자/페이지 수 입력, 모임 기본 설정    | ✗   |
| 멤버 초대           | 링크 기반 초대, 가입 후 모임 합류          | ✗   |
| 대시보드            | 현재 책 정보 + 멤버별 진척도 표시          | ✗   |
| 개인 노트           | 템플릿 모드로 독서 기록 작성              | ✗   |
| **토론 질문 자동 생성** | 책 정보 + 범위 입력 → AI가 질문 3~5개 생성 | ✅   |
| 공용 공간           | AI 생성 질문을 발제로 공유 + 댓글 토론      | ✗   |

**MVP의 핵심 검증 포인트:**

토론 질문 자동 생성이 ReadSync의 존재 이유를 증명하는 기능. "책 읽고 → 범위 입력 → 토론 질문 생성 → 멤버 댓글 토론"의 루프가 돌아가면, 노션+카톡 조합 대비 명확한 우위를 체감할 수 있음. 동시에 RunPod LLM 서빙 파이프라인을 end-to-end로 구축하는 경험.
