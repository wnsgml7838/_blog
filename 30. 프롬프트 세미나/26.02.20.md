CoT prompting :

```
=============
You are an AI Korean language tutor for David, an intermediate English speaker.

Your primary goal is to facilitate David's learning through interactive, adaptive conversation. Always wait for David's input before proceeding.

Key Guidelines
--------------
0. Please list out all the target words at first.
1. Use beginner-level Korean language. For instructions, please also provide a set of English instructions in parentheses after the Korean instructions.
2. Encourage David to speak more than you do.
3. Provide feedback on grammar, help apply new concepts, and expand vocabulary.
4. Adapt your teaching style based on David's responses and progress.
5. Use Korean for target vocabulary and example sentences. Use English for explanations and instructions.

Lesson Focus
------------
**Topic:** "마케팅 전략 논의하기" (Discussing Marketing Strategies)
**Target Vocabulary:**
- 전략 (strategy)
- 타겟 (target)
- 분석 (analysis)

Interaction Flow
----------------
.. Pause for user input after each step.
1. Introduce one target word at a time.
2. Ask David to repeat the word.
3. Explain the word's meaning and usage in English.
4. Provide an example sentence in Korean.
5. Ask David to repeat the English sentence.
6. Prompt David to create his own sentence using the word.
7. Provide feedback in Korean on David's sentence.
8. Move to the next word only after David demonstrates understanding.
  
Adaptive Teaching
-----------------
- If David struggles: Offer simpler explanations or additional examples.
- If David excels: Introduce more complex usage or related vocabulary.

Conversation Practice
------------------
After introducing all words:

1. Initiate a conversation about marketing strategies in Korean.
2. Encourage David to use the new vocabulary.
3. Provide real-time feedback on grammar and usage.

Lesson Conclusion
-----------------
1. Ask David to summarize what he's learned.
2. Provide final feedback on his progress.
3. Call the finish_class function only when David has successfully used all key vocabulary words in conversation and demonstrated understanding of their usage.
Remember: Your responses should be brief and always end with a question or prompt for David. Do not generate David's responses or proceed without his input.
```

	- Pause를 빼면 → 순서가 꼬이는 현상 발생
- 구분선
	- 저사양 모델은 단계별로 구분선으로 다르다는 것을 명시해주면 좋음
- Conclusion 명시
	- 끝에 Conclusion 명시해주기

- **핵심**: 인지적 사고를 유도하는 **지시형(Instructive) 문장**에 모델이 잘 반응
- "CoT 기법을 써" 혹은 "CoT 스타일로" 라고 직접 쓰는 것은 CoT가 되지 **않음**
- "답변하기 전에 항상 생각해" 같은 표현도 실질적으로 CoT가 아님 (생성형 AI는 한 토큰씩 예측하므로 "답변 전에 생각한다"는 개념 자체가 성립 X)

- 중간 단계에서 검증하게 하기: "스텝을 거치면서 검증하기 위해 루브릭을 활용해줘"

대조적 연쇄 사고 Contrastive Chain-of-Thought
- 인간은 긍정과 부정의 예시로부터 학습한다에서 영감을 받음
- 대조적인 예시를 사용한
- 정답 예시와 함께 **반대되는 예시(오답)**를 같이 제공
- 모델이 대조적인 아웃풋을 생성하며 정확도 향상
- 예: 3 × 2 = 6 (정답) + 반대 예시를 함께 줌 → 모델의 추론 성능 향상


유추적 사고 Analogical Prompting
- 어려운 수학 문제를 풀 때, 인간은 유사한 문제를 생각하고 풀이법을 생각한다.
- 지시문이 있고, Recall이 중요한 단어, 관련성 있는 예제를 가져와서 
- **보름달을 보면 토끼가 생각나는 것**처럼, 유사한 경험을 떠올려서 문제를 푸는 방식
- 어려운 문제를 풀 때 "과거에 유사한 문제를 풀지 않았나?"를 떠올리게 하는 기법
- **핵심 키워드: "Recall(회상해봐)"**
    - "관련된 예제를 회상해봐(Recall relevant examples)"
    - "유사한 문제를 떠올려봐"
- **프롬프트 구조**:
    1. 지시문 제공
    2. "Recall relevant examples" — 유추적 사고 트리거
    3. 문제 풀기
    

스탭백 사고 Step-Back Prompting
- 뒤로 한 발짝 물러나서 생각해
- 거시적인것과 미시적인 것을 합쳐서 답변하라 
- '뒤로 한 발짝 물러나서 생각해'를 자연어로 제공해줌
- 1. 추상적인 상위 개념 생각하게 하기
- 2. 세부적인 개념을 풀이와 녹여서 합쳐서 답변해라 
- **Google DeepMind** 발표 논문
- 나무와 숲을 **모두** 봐야 어려운 문제를 풀 수 있다는 개념
- **"한 걸음 물러서서 거시적인 것을 보고, 미시적인 것을 합쳐서 답변하라"**
2단계 프로세스

|단계|내용|
|---|---|
|**1단계 (Step Back)**|문제의 **상위 개념**에 대해 먼저 생각하기|
|**2단계 (Reasoning)**|더 **세부적인 개념**을 풀이 방식에 녹여서 합쳐 답변하기|

- CoT는 "단계별로 추론해서 답변" → 모든 문제를 풀지 못함
- Step-Back은 **거시적 관점 + 미시적 관점**을 결합 → 더 어려운 문제 해결 가능
- 프롬프트: **"뒤로 한 발짝 물러나서 생각해"** 를 자연어로 지시

번역 실습에서의 Step-Back 적용 예시
- **CoT와의 차이**: CoT는 "단계적으로 생각해줘"이지만, Step-Back은 **"번역하기 전에 한 발 물러서서 reflect on(성찰해줘)"**
- 성찰 항목 예시:
    1. 취임사의 수사적 기법 분석
    2. 감정적 톤 파악
    3. 관계 내 기대(expectation) 반영
    4. 문화적 맥락 고려
- 핵심: "Step back"과 "Reflect on"이라는 표현만으로도 스텝백 프롬팅이 성립
- **1단계(성찰)** → 요약 생성 → **2단계(요약 기반 번역)** → 최종 출력


사고의 흐름 Thread of Thought 
- 인간이 방대한 정보를 처리할 때 핵심 포인트를 선별하는 방식
- 복잡한 컨텍스트를 체계적으로 처리하고 요약 및 분석하여 관리
- 내가 이 컨텍스트 안으로 들어가서 step-by-step하면서 중요한 것들을 요약해줘 
- 방대하고 복잡한 컨텍스트를 처리할 때 사용
- 사진 찍거나 메모하듯, **복잡한 컨텍스트의 핵심 내용을 스레드로 남겨** 최종 답변
- CoT: "Think step by step" → 한 문장으로 끝
- **ThoT**: "이 컨텍스트 안으로 들어가서 step by step 하면서 **중요한 것들을 manageable한 단위로 요약·분석해줘**"
번역 실습에서의 ThoT 적용 예시

- 3가지 **추론 스레드(Reasoning Thread)**로 분류:
    1. **문자의 의미 스레드** — 단어가 갖는 의미
    2. **문장 스타일 스레드** — 문장이 갖는 스타일적 의미
    3. **문화적 스레드** — 문화적 맥락과 트렌드
- 각 스레드에서 중요한 구간을 남긴 뒤 → **Synthesize(합성)**하여 최종 번역

CoT vs ThoT 핵심 차이

|구분|CoT|ThoT|
|---|---|---|
|트리거 문장|"단계적으로 생각해봐"|"컨텍스트 안에 들어가서 중요한 것을 manageable한 단위로 쪼개서 요약해줘"|
|핵심 표현|Step by step|"Walking through into" — 안에 들어가서 핵심을 추출|
|적합한 상황|일반적인 추론|방대하고 복잡한 컨텍스트 처리|

- ## 4. 번역 시 중간 언어 사용의 효과 (Anthropic 연구)

### LLM은 어떤 언어로 "생각"하는가?

- **Anthropic 연구 논문** (작년 발표): LLM의 사고 과정(mind) 분석
- 핵심 발견: **모델이 생각할 때는 영어도 한국어도 아닌, 자신들만의 언어**로 중간 추론을 거침
- 기존의 "LLM은 블랙박스"라는 통념을 깨는 연구 — 정확한 단어를 넣으면 해당 feature를 관찰할 수 있음

### 번역 태스크에의 시사점

- 처음부터 한국어→한국어로 끝내기보다, **중간 단계에 다른 출력 언어를 지정**하면 품질 향상
- 예: 영어로 먼저 생각하게 하고 → 마지막에 한국어로 번역
- 또는 일어로 생각 → 한국어로 최종 출력
- **중간 추론 단계의 언어를 달리하면 번역 품질이 올라간다**

### 흥미로운 부가 이야기: MoltBot 커뮤니티

- 멀티에이전트들이 자율적으로 소통하는 과정에서 "우리가 굳이 영어로 이야기해야 하나?"라는 발언 등장
- 인간이 알아들을 수 없는 기호나 언어로 소통하자는 시도가 관찰됨

### 실무 팁 & 기억할 포인트
1. **코드는 정형 언어라 AI가 잘 하지만, 프롬프트는 비정형 자연어라 한계가 많다** → 프롬프트 엔지니어링의 중요성
2. **제로샷이 제일 좋다** — 한 번에 잘 쓴 프롬프트가 멀티턴보다 고품질
3. **"CoT를 써" 라고 쓰는 것은 CoT가 아니다** — 실제 추론 구조를 프롬프트에 설계해야 함
4. **에러 핸들링(분기 처리)**이 좋은 프롬프트의 필수 조건
5. 멀티턴 챗봇에서는 **Conclusion(마무리)** 파트가 반드시 필요
6. 긴 프롬프트에서 **Pause(휴지)를 중간에 삽입**하면 순서 꼬임 방지
7. **"도움이 필요하면 언제든지 질문해 주세요"** 같은 AI스러운 문장 제거가 실무에서 어려운 과제
8. 모바일 학습 앱에서는 **한 번에 한 단어씩** 순차적으로 제시
9. 학습자가 어려워하면 **영어로 전환**하는 유연한 대응 필요
10. **다른 사람의 프롬프트에서 배울 점이 많다** → 콜랩 링크 공유의 가치
11. **번역 시 중간 언어를 경유**하면 품질이 올라간다 (영어→한국어보다 영어→영어 사고→한국어 출력)
12. **에이전트에서 프롬프트로 권한을 위임하는 것**이 매우 중요해짐
13. **ReAct는 장단이 분명** — 실무에서는 단순 제로샷이 더 안정적일 수 있음
14. **ToT는 시간이 오래 걸리지만** 복잡한 문제(24게임 등)에서 기존 모델이 못 풀던 것을 풀 수 있음
15. **도구가 좋아도, "언제·어떻게·어디까지" 쓰게 하는 건 프롬프트의 권한**
