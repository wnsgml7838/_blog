**문제**: 의미없는 짧은 트랜스크립트에도 리더 피드백이 생성됨

- 스코어링 결과가 존재하면 (모두 0점이라도) 피드백 LLM이 호출됨
- **해결**: `generate_feedback.py`에 LLM gate 추가 → 의미없는 transcript이면 피드백 스킵

**1. general_analysis가 `null`로 빠짐**

- `mapped_transcript`가 비어있지 않아서 (1개 utterance) "음성 미감지" 하드코딩 응답을 안 탐
- LLM 호출 → 실패 → `except`에서 `general_analysis`를 설정하지 않음 → `null`
- 에러 12번 반복 (비용 + 시간 낭비)

**2. 스코어링이 의미없는 데이터에도 전부 실행됨**

- "날씨였습니다." 한 문장에 30개 항목 평가 → 29개 0점
- LLM 호출 비용 + 시간 낭비
- DB에 기록해도 장기 패턴 분석에 쓸 가치 없음

**3. gate가 피드백 앞에만 있음 → 사각지대**

- gate가 `generate_feedback` 직전에만 있으므로, 그 앞의 `analyze_general` + `scorer`는 여전히 의미없는 transcript에 대해 풀로 실행됨
- gate를 `map_speakers` 직후로 올리면 3개 노드 모두 스킵 가능
