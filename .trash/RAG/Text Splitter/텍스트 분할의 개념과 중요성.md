- 한 페이지를 통으로 넣어서 문서 검색을 사용하지 않음
- 한 페이지도 잘게 쪼개서 
왜 잘게 쪼개야할까?
- 한 페이지 내에서 답변에 필요한 내용이 처음부터 끝까지 다 필요하지 않음
- 논문 같은 페이지를 보더라도 사용자가 질문을 했을 때 질문에 대한 답변이 처음부터 끝까지 다 중요하지 않잖아

**분할의 필요성** 
- 1. LLM 인풋 토큰에 제한에 걸리지 않기 위해서 필요한 정보만 압축해서 넣어주기 위해
- 2. 페이지를 통으로 넣어서 찾는것과 필요한 정보만 발라내서 인풋으로 넣어줬을 때 답변 품질의 차이가 많이 남 (할루시네이션 등)
	- 그래서 효율적으로 잘라내는 전략을 먼저 세워야함
	- 나중에 발라내진 덩어리(청크)를 유사도 검색을 할건데(질문이랑) 질문에 대한 답변하기 위해서 청크가 얼마나 중요한지 계산을 하는 것 -> 청크를 만들 때 잘 만들면 유사도도 높게 잘 나올것임 
	- 청크를 잘못 나눠놨으면 유사도 검사할 때 잘 안되고 -> 불필요한 정보들이 같이 검색되서 나옴 

**문서분할 과정**
1. 문서 구조 파악 : PDF 파일, 웹 페이지, 전자 책 등 다양한 형식의 문서에서 구조를 파악합니다. 이는 문서의 헤더, 푸터, 페이지 번호, 섹션 제목 등을 식별하는 과정을 포함할 수 있습니다.
2. 단위 선정 : 문서를 어떤 단위로 나눌지 결정합니다. 이는 페이지별, 섹션별,또는 문단별일 수 있으며, 문서의 내용과 목적에 따라 다르다.
3. 단위 크기 선정(chunk size) : 문서를 몇 개의 토큰 단위로 나눌 것인지를 정한다
4. 청크 오버랩(chunk overlap) : 분할된 끝 부분에서 맥락이 이러질 수 있도록 일부를 겹처서 분할하는 것이 일반적
